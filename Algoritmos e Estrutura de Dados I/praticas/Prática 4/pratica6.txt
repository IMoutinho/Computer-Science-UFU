(a) Explique porque a estrutura nó, para a criação de lista encadeada, é definida recursivamente.
    A estrutura no é definida recursivamente como prox porque no caso de lista encadeada, a celula de memória guarda não somente
    o dado, ou valor que será atribuido, mas também guarda um ponteiro que aponta para o próximo elemento e se chama recursivamente. 
    Ele é definido dessa forma pois sempre que for inserido um novo elemento, este elemento precisa apontar para o elemento seguinte e
    assim sucessivamente.

(b) Indique vantagens da estrutura de encadeamento de nós para a criação de listas.
    As vantagens são principalmente voltadas para quando serão listas que terão atualização de dados com frequencia,
    pois é mais fácil inserir e remover itens da lista. Além disso, é uma forma mais segura 
    de guardar os dados e evitar apagamentos acidentais, pois não deixo exposto o local ou variavel aonde está sendo guardado 
    determinado valor. 
    Além disso, ela pode guardar diferentes tipos de dados e é mais flexivel neste quesito. 
    Ela também tem seu tamanho definido dinamicamente, o que permite que ela cresça e diminua durante a execução do programa, evitando
    assim o disperdicio de memória que não será utilizada.

};

// Definição do tipo No
typedef struct no No;

// Função para criar uma lista vazia

   return NULL;

// Função para inserir um elemento na lista
No* insereInicio(No *l, int elem){
  No* novo;
  novo = (No*) malloc (sizeof(No)); 
  if (novo == NULL) return novo;

  novo->item= elem;
  novo->prox = l;
  return novo;
}

int main(){
 No *lnum; // lista de números com encadeamento de nós

 for(int i=0;i<10;i++){  //  inserir 10 elementos    
    scanf("%d",&val);
    lnum = insereInicio(lnum, val);
 }
}
